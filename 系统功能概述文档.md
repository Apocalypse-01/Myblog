# 个人笔记系统（MyBlog）功能概述文档

## 一、系统概述

本系统是一个基于前后端分离架构的个人笔记与博客管理平台，旨在为个人用户提供一个集内容创作、分类管理、社交互动、多格式导出于一体的知识管理工具。系统前端采用 **Vue 3 + TypeScript + Vite** 技术栈，后端采用 **Spring Boot + MyBatis + MySQL**，通过 **JWT 无状态认证**保障数据安全，使用 **BCrypt 加密算法**保护用户密码。

### 1.1 系统技术架构

| 层次           | 技术选型                     | 说明                  |
| ------------ | ------------------------ | ------------------- |
| 前端框架         | Vue 3 (Composition API)  | 组件化开发，响应式数据绑定       |
| 前端语言         | TypeScript               | 类型安全，减少运行时错误        |
| 构建工具         | Vite                     | 快速冷启动与热模块替换         |
| UI 组件库       | Element Plus             | 统一的 UI 设计语言         |
| Markdown 编辑器 | md-editor-v3             | 支持实时预览的 Markdown 编辑 |
| 状态管理         | Pinia                    | Vue 3 官方推荐状态管理方案    |
| 路由管理         | Vue Router 4             | SPA 前端路由            |
| 后端框架         | Spring Boot 3            | Java 企业级开发框架        |
| ORM 框架       | MyBatis                  | SQL 与对象映射           |
| 数据库          | MySQL 8.0                | 关系型数据存储             |
| API 文档       | Knife4j (Swagger)        | 自动生成 RESTful API 文档 |
| 认证机制         | JWT (auth0-java-jwt)     | 无状态用户认证             |
| 密码加密         | BCrypt (Spring Security) | 单向哈希加密              |
| HTTP 客户端     | Axios                    | 前端 HTTP 请求封装        |

### 2.1 用户模块（User Module）

#### 功能说明

| 子功能    | HTTP 方法 | API 路径                 | 权限  | 说明                    |
| ------ | ------- | ---------------------- | --- | --------------------- |
| 用户注册   | POST    | `/user/register`       | 公开  | 邮箱注册，BCrypt 加密密码      |
| 用户登录   | POST    | `/user/login`          | 公开  | 验证凭证，返回 JWT Token     |
| 获取个人信息 | GET     | `/user/getUserInfo`    | 需登录 | 根据用户 ID 获取信息          |
| 更新个人信息 | PUT     | `/user/updateUserInfo` | 需登录 | 修改姓名、手机、签名等           |
| 上传头像   | POST    | `/user/uploadAvatar`   | 需登录 | 支持 JPG/PNG/GIF，最大 2MB |

#### 前端校验规则

# **邮箱**：正则 `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`，实时校验，不合法时显示红色提示

- **手机号**：正则 `/^1[3-9]\d{9}$/`，11 位中国大陆手机号格式
- **密码**：非空校验，使用 `show-password` 组件支持密码显隐切换

### 2.5 辅助功能模块（Auxiliary Module）

#### 2.5.1 多格式文章下载

| 格式       | HTTP 参数           | 文件类型    | 说明        |
| -------- | ----------------- | ------- | --------- |
| Markdown | `format=markdown` | `.md`   | 保留原始排版格式  |
| 纯文本      | `format=txt`      | `.txt`  | 去除格式的纯净文本 |
| EPUB 电子书 | `format=epub`     | `.epub` | 支持电子书阅读器  |

下载流程支持进度显示、取消操作、失败重试。

#### 2.5.2 草稿暂存

利用 `localStorage` 实现编辑器内容的本地暂存：

- **保存草稿**：将标题、标签、内容序列化后存入 `localStorage`
- **恢复草稿**：编辑器初始化时自动检测并回填草稿数据
- **编辑模式**：支持文章编辑时从服务端拉取原文内容，编辑后提交更新

## 三、关键业务流程

### 3.1 用户注册流程

```
用户输入 → 前端校验 → 发送 POST /user/register → 服务端校验邮箱唯一性
→ BCrypt 加密密码 → 写入 user 表 → 返回结果 → 提示注册成功 → 跳转登录页
```

### 3.2 用户登录流程

```
用户输入 → 前端校验 → 发送 POST /user/login → 服务端验证 BCrypt
→ 生成 JWT Token → 返回用户信息 + Token → 前端存储 localStorage
→ 更新 Pinia 全局登录状态 → 跳转首页
```

### 3.3 文章发布流程

```
进入写作页 → Markdown 编辑器输入 → 选择标签 → 可选：保存草稿(localStorage)
→ 点击发布 → 前端校验(标题/标签/内容非空) → PUT /article/release
→ 服务端写入 article 表 → 返回 articleId → 询问是否查看
→ 是：跳转文章详情页 / 否：留在编辑器
```

### 3.4 文章阅读与互动流程

```
浏览列表页 → 选择分类标签(全部/日常/技术/杂谈/其他) → 分页加载
→ 点击文章标题 → GET /article/getArticle → 渲染 Markdown 内容(MdPreview)
→ 点赞/收藏操作 → 校验登录状态 → 发送请求 → 更新 UI 状态
→ 评论操作 → 输入评论内容 → 发表 → 刷新评论列表
→ 下载操作 → 选择格式(MD/TXT/EPUB) → 进度条显示 → 完成/取消/重试
```

### 3.5 JWT 认证拦截流程

```
请求到达 → JwtInterceptor.preHandle()
→ 检查请求路径是否在白名单(注册/登录/Swagger/公开接口) → 是：放行
→ 否：获取请求头中的 token
→ token 为空 → 返回 401 未授权
→ token 非空 → JWT 解析验证 → 成功：放行 / 失败：返回 401
```

**图片类型**：数据库 ER 图\
