📌 分层设计原则

各层职责明确，上层依赖下层，下层不感知上层业务。
层与层之间通过接口通信，降低耦合便于维护扩展。

📌 前后端分离

前端专注界面交互，后端专注业务处理。
两者独立开发、独立部署，通过 HTTP/JSON 通信。

📌 数据流示例（用户登录）

① 前端提交表单 → ② Controller 接收 → Service 调用 Mapper →
③ Mapper 查询数据库 → ④ MySQL 返回记录 →
② BCrypt 比对 → JWT 生成 Token → ① 前端存储并更新界面

📌 安全横切

JWT 工具类统一生成/解析令牌，拦截器在请求进入 Controller 前校验。
白名单放行注册/登录/公开查询，其余接口需携带有效令牌。
