二、关系类型详解

┌──────────────────┬──────────────┬──────────────┬───────────────┐
│      子表        │    父表      │   关系类型    │   基数约束     │
├──────────────────┼──────────────┼──────────────┼───────────────┤
│ article          │ user         │ 多对一 (N:1)  │ 多个文章属于   │
│                  │              │              │ 同一用户       │
├──────────────────┼──────────────┼──────────────┼───────────────┤
│ article_comment  │ article      │ 多对一 (N:1)  │ 多条评论属于   │
│                  │              │              │ 同一文章       │
├──────────────────┼──────────────┼──────────────┼───────────────┤
│ article_comment  │ user         │ 多对一 (N:1)  │ 多条评论由     │
│                  │              │              │ 同一用户发表   │
├──────────────────┼──────────────┼──────────────┼───────────────┤
│ article_like     │ article      │ 多对一 (N:1)  │ 多条点赞属于   │
│                  │              │              │ 同一文章       │
├──────────────────┼──────────────┼──────────────┼───────────────┤
│ article_like     │ user         │ 多对一 (N:1)  │ 多条点赞由     │
│                  │              │              │ 同一用户发起   │
├──────────────────┼──────────────┼──────────────┼───────────────┤
│ article_favorite │ article      │ 多对一 (N:1)  │ 多条收藏属于   │
│                  │              │              │ 同一文章       │
├──────────────────┼──────────────┼──────────────┼───────────────┤
│ article_favorite │ user         │ 多对一 (N:1)  │ 多条收藏由     │
│                  │              │              │ 同一用户发起   │
└──────────────────┴──────────────┴──────────────┴───────────────┘

说明：本系统所有关系均为"多对一（N:1）"，不存在多对多（M:N）关系。
      每个关联表（article_like、article_favorite）自身即为 user 与 article
      之间的关联中间表，但各自独立存在，不构成传统意义上的 M:N 拆表。

  · avatar      — 头像URL，VARCHAR(255)

六、冗余字段说明

本系统在以下两张表中采用了数据冗余设计：

┌──────────────────┬──────────────────┬────────────────────────────────────┐
│      表名         │     冗余字段      │              设计目的               │
├──────────────────┼──────────────────┼────────────────────────────────────┤
│ article          │ author_name      │ 避免查询文章时JOIN user表获取作者名  │
├──────────────────┼──────────────────┼────────────────────────────────────┤
│ article          │ likes            │ 避免每次展示文章时COUNT点赞表       │
├──────────────────┼──────────────────┼────────────────────────────────────┤
│ article          │ favorites        │ 避免每次展示文章时COUNT收藏表       │
├──────────────────┼──────────────────┼────────────────────────────────────┤
│ article_comment  │ user_name        │ 避免查询评论时JOIN user表获取用户名  │
└──────────────────┴──────────────────┴────────────────────────────────────┘

冗余字段的维护规则：
  · author_name / user_name：在创建记录时写入，用户改名后不追溯更新
  · likes / favorites：在点赞/收藏/取消操作时由应用层代码同步 ±1
