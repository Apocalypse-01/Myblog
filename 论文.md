摘要伴随各类视频的广泛流行，人们已然习惯于将自身的想法、知识记录下来，但是因为知识的碎片化，导致对这些知识进行整理的需求也持续递增，这使得针对不同时间、不同场景的知识管理成为越来越多的人的必需之物。因此，本系统旨在构建一个功能完善、使用便捷的智能笔记系统。
本系统具备几个方面的优势。其一，数据自主可控。全部笔记内容存储于用户自身的数据库、服务器内，不会因第三方平台政策的变动或者服务的停止运营而受到影响。用户对数据享有所有权、支配权。其二，功能完备。智能笔记系统拥有笔记撰写、分类管理、全文搜索、多格式导出、评论互动、以及数据显示、点赞收藏等重要功能。其能够支持从内容创作一直到知识沉淀的整个流程，而且界面简洁，操作直观，数据简洁。技术实践方面，智能笔记系统从前端界面、后端服务到数据持久化，运用了Web开发的主流技术。比如Vue3的组件化与响应式机制、SpringBoot的RESTful接口设计、MyBatis的对象关系映射、JWT的无状态认证、BCrypt的加密存储。从理论上来说，智能笔记系统可为Web全栈开发提供技术实践上的参考。在实际使用中，智能笔记系统可为个人创作者建立安全、高效、好用的内容管理平台。智能笔记系统所具备的技术优势、其功能得以实现的情况，对于推动Web开发技术向前发展、促进知识的分享而言，都有着一定的意义。

 Design and Implementation of an Intelligent Note-Taking Application Based on Spring Boot
Abstract With the widespread popularity of various video formats, people have become accustomed to recording their thoughts and knowledge, and the demand for organizing this information continues to grow. As a result, knowledge management tailored to different times and contexts has become an essential need for an increasing number of individuals.
This system offers several advantages. First, data is fully self-controlled and secure. All note content is stored within the user's own database and server, ensuring it remains unaffected by changes in third-party platform policies or service discontinuation. Users retain full ownership and control over their data. Second, the system features comprehensive functionality. The intelligent note-taking system supports essential functions such as note creation, categorization, full-text search, multi-format export, comment interaction, data visualization, likes, and bookmarking. It enables a complete workflow from content creation to knowledge consolidation, with a clean interface, intuitive operation, and streamlined data management. Technically, the system leverages mainstream web development technologies across its frontend, backend, and data persistence layers—such as Vue 3’s component-based architecture and reactivity, Spring Boot’s RESTful API design, MyBatis for object-relational mapping, JWT for stateless authentication, and BCrypt for encrypted storage. In theory, this system provides practical reference for full-stack web development. In practice, it empowers individual creators with a secure, efficient, and user-friendly content management platform. The technical strengths and functional capabilities of the intelligent note-taking system hold significant value in advancing web development technologies and promoting knowledge sharing.
Key words note-taking platform; front-end and back-end separation; Vue 3; Spring Boot; Markdown editor; JWT authentication
 
第1章绪论
1.1	项目背景与意义
随着当今信息量持续增长，短视频与碎片化学习变为常态[1]，人们对于知识的记录和整理的想法正在不断的加深，对于随时笔记的需求也更加深入。但是因为笔记的碎片化而导致对于知识的整理会成为一个麻烦[2]。
在这样的环境下，个人笔记慢慢不再只是随手记事的工具，而是逐渐承担起知识整理、思考记录乃至内容输出的任务。但是，对于目前市面上的笔记类产品，本地软件虽然响应快、数据在本地，但是假如换一台设备就不能随时查看了。云端平台虽然方便，但是数据并不存储在本地就可能会导致数据出现丢失，而且有的云端平台会对内容做出一些限制，并且隐私方面也会出现一定担忧，所以安全、自主、可控这三个方面似乎很难在一款产品里同时满足，这便是成为了本课题研究和开发的切入点。
从实际应用价值来看，自己建立一套自己的笔记系统，最大的好处便是可以对数据进行完完全全的掌控[2]。所有笔记内容存储在自己的数据库和服务器上，不会因为其他平台的而遭受到限制，也不会因为突然停止运营而导致数据丢失。还有一个优点就是页面的高度可定制化，功能可以随便添加，页面可以随意定制，甚至不仅可以记笔记，也能开发一个全能平台。
从技术角度看，本系统的开发经历前后端和数据库的开发与和实现，覆盖基本的Web开发所需要的所有功能。前端单独写页面，后端注重逻辑的实现，MySQL针对数据进行存储，前后端通过HTTP/HTTPS协议+JSON格式进行通讯，前端利用Axios发送请求，后端使用RESTful API用来接受请求。
对于个人使用体验来说，一个干净、不卡手、功能不杂的笔记工具，的确会让人更愿意打开它写点东西。不仅是干净，而且界面也是符合自己审美的，看着舒服，写东西当然也舒服了[4]。当然本系统也能多人共用一个服务器，数据库相当于是共享的，但是做了用户的区分，还是有一定的用户权限管理的。系统里含有点赞、收藏和评论这类互动的功能，不算重要，但是基本能够使用了。
本文对开发一个可以进行自部署的智能笔记系统进行了整体分析，该系统解决了市面上现成方案在数据自主性和个性化方面存在的不足，也为学习当前主流前后端技术提供了一个合适的实践载体，并且可以在日常使用中持续进行优化和迭代，属于一项具有长期价值的工作。
1.2	国内外研究现状
1.2.1	国外研究现状
国外在个人知识管理和笔记工具起步的早。经过多年发展，已经形成了几条很清晰的路线。常用的结构化云端笔记是Notion和Evernote，这两个名字最常被提到。Notion将文档、数据库、看板全放在一起，用Block编辑器像拼积木一样搭建自己的工作空间。Evernote的关注点不一样，Evernote更在意怎么收集信息并快速搜索，全文搜索和标签功能已经成熟。Notion和Evernote共同点是数据都在云上，只要有网络就能访问。但这也意味着在没有网络时会不方便，因为数据没有保存在本地。还有一种工具，专门给开发者和重度用户使用，Obsidian和Logseq就是这类。Obsidian把本地的Markdown文件当作底层，Obsidian在笔记之间建立双向链接，Obsidian还能生成知识图谱。笔记逐渐织成网络。Obsidian适合长期积累内容、反复查看。Logseq采用开源方式，Logseq存储同样使用Markdown文件，Logseq的编辑方式改成大纲式，一行一行展开。Logseq认为平台不该锁住数据，Logseq想迁移，随时可以。这类工具好处很明显：数据全在自己硬盘上，没有平台关闭就丢失的风险，也没有私有格式的锁住。

这几年，我看到国内个人笔记和知识管理发展很快。出现了几种形态。第一种是服务平台。比如有道云笔记、语雀、飞书文档，有道云笔记、语雀、飞书文档各自占有一席。有道云笔记、语雀、飞书文档背后都有大厂支撑，更新快，系统很少出错。免费版功能常受限，商业化一直在影响免费版功能。我常用有道云笔记、语雀、飞书文档，感受有道云笔记、语雀、飞书文档的更新速度。有道云笔记、语雀、飞书文档变化明显。

另一类则是使用用别人发布的开源框架写的。在国内有名开源框架便是Hexo和Halo。两种框架侧重不同，Hexo属于静态博客方案，适合无需后台操作的纯内容展示场景；Halo则是更加成熟的动态博客系统，与本系统在技术思路上有相似之处但更为庞大[3]。

第2章相关技术介绍
2.1	系统开发平台
2.1.1	IntelliJ IDEA
本项目使用的后端平台是IDEA。IntelliJ IDEA是一款广受欢迎的集成开发环境（IDE），专为Java编程语言设计，但也支持其他语言如Kotlin、Scala和Groovy。由JetBrains公司开发，该公司以其严谨的东欧程序员团队而闻名。
IntelliJ IDEA最强大的功能之一是其智能代码补全。与其他编辑器相比，IDEA的补全不仅基于代码上下文，还可以识别到类层次结构，以及能推断最可能使用的类或方法。这种深度的补全大幅度减少了代码编写的时间，尤其在大型项目中表现尤为突出。虽然本系统并不是什么大型项目，但是其代码自动补全依旧为日常代码的编写提供了巨大的帮助。
此外，IntelliJ IDEA也提供了实时的代码分析与错误提示。对比VSCode等轻量编辑器，IDEA的代码分析能力非常的强，在代码编辑过程当中能够提前检测到潜在的错误，并进行提示，能够帮助开发者在编写时就避免一些常见的错误，从而减少了调试的时间，加快开发进度。对于本系统帮助不可谓不大。
编写代码的过程中，调试和测试毫无疑问在开发过程中占据了很大一部分时间，IntelliJ IDEA通过提供无缝的调试工具，简化了这一过程。IDEA的调试器支持对代码的断点设置、变量监控、内存分析等功能。相比轻量编辑器的调试功能，IntelliJ IDEA的集成更加深入和强大。调试器功能强大，在开发过程中，本系统通过该调试器找到和解决了不少bug。
总的来说，IntelliJ IDEA通过其智能的代码检测补全功能、强大的重构功能、一体化的开发环境以及其丰富的插件生态，成为了本次系统开发的首选后端工具。
2.1.2	VisualStudioCode
我使用VScode编辑器来进行前端的编写。VSCode是微软公司出的轻量、跨平台、开源编辑器，启动快，插件多，已经成为前端开发里最常用的工具。使用Vue3时，VSCode加上Volar扩展能提供单文件组件的语法高亮、智能补全和类型检查，基本满足开发需要的语言服务。
在实际工程里，VSCode集成终端直接提升开发体验。开发者直接在终端敲npm run dev，就能启动Vite开发服务器；当编译出错时，错误信息点一下就跳到对应源码行，省去在编辑器和外部命令行之间来回切换的麻烦。Vue3前端开发经常需要快速验证样式和交互。改一段CSS，或者改一个组件，刷新浏览器马上看到效果。这样的高强度、快节奏工作里，VSCode用的内存少，启动快，比IntelliJ IDEA那种重量级IDE更轻巧、更专注。于是我把VSCode选为本项目前端开发的核心编辑工具，它在提升编码效率和调试便利方面表现突出
2.1.3	MySQL数据库
我选择MySQL作为后端的持久化数据库。MySQL是Oracle维护的开源、稳定、跨平台的关系型数据库，已经在行业里用了好多年，Web开发里有很多真实的生产案例。技术上，SpringBoot搭配MyBatis本来就很配MySQL，文档和社区也很活跃，遇到配置或性能问题时，有很多现成的案例可以直接参考，这样对日常开发和后期维护都很方便。
在数据库技术和功能方面，MySQL拥有完整的关系数据库能力。MySQL的核心功能分四点。第一，MySQL支持标准的SQL，方便我们写复杂查询和聚合；第二，默认的InnoDB存储引擎实现ACID事务，保证高并发写入时数据一致；第三，MySQL自带多版本并发控制，保持读写性能的同时减少行锁冲突；第四，MySQL提供索引优化、视图、触发器和存储过程等高级特性，留出空间让我们以后调优和扩展功能。
在这个项目里，MySQL负责把所有业务数据保存下来。用户表、文章表、评论表和点赞收藏表都在MySQL里设计和存放。前端和后端通过应用程序接口来读写数据，数据库连接不直接给前端看，这样数据更安全，也更好管理。因为MySQL能保证数据可靠，MySQL也让以后维护更容易。

第6章系统测试
6.1	系统测试的概念
在软件开发的完整过程中，系统测试是一个绕不开的关键阶段，也是评判软件质量到底过不过关的主要凭据。它的做法是把已经拼装完成的软件连同硬件设备、数据库、网络环境等周边要素凑在一起，放在接近真实使用的条件下跑一遍，从功能是否齐全、跑起来是否流畅、安全方面有没有漏洞等多个角度去做一次全面的检查。所以在开发的时候就需要用不同的手段反复去检验，既要有合理的测试安排，也要抓准时机，尽早把隐患挖出来处理掉。等到开发方这边把系统层面的测试和问题修复都过完一遍之后，最后还有一个验收测试的环节，由需求方或者最终用户来把关，看看软件到底符不符合当初定下的标准和实际使用的要求，这一步既是对软件负责，也是给用户一个交代。
按照软件测试的常规流程，测试工作的开展顺序依次为单元测试、集成测试、功能测试和验收测试，每一层分别担着不同的任务。
单元测试是第一步，由开发人员自己来编写数据，确定功能逻辑是否正常。它聚焦在代码层面最小的那些单元——一个函数、一个方法或者一个类，对着内部的逻辑分支逐个去走，看看每个小模块单独跑的时候有没有出错的地方，用的是白盒的路子。这一步的目的是保证每个零件本身是好的。
所有功能模块都开发完了之后，就进入集成测试。这个阶段不再盯着单打独斗的模块看，而是把它们按照设计拼到一起，重点查模块之间对接的接口通不通、数据传过来是不是对的、几个模块凑一块能不能正常配合。很多问题是在单模块测试时看不出来的，只有拼起来才会暴露，集成测试要解决的正是这类问题。
再往下是功能测试。这一步换了个思路，用的是黑盒的方式——不看代码里面的实现，只站在用户的立场上，对照着需求说明书一项一项去核对系统有没有把该做的功能都做出来、做得对不对。功能测试不管后台是怎么实现的，只看前台的表现符不符合当初的预期。
单元测试、集成测试和功能测试都过了以后，最后是验收测试。这个阶段通常由最终使用者或者客户来主导，依据实际的业务场景和事先签好的约定，对整个系统做一次综合评定，看看到底能不能在日常工作中用起来、各项功能是不是真的贴合实际需要，最终决定软件是否达到了交付的条件
通过上述系统性且有序的测试流程，能够全面、深入地对系统的质量与可靠性进行检验，确保软件交付后能够稳定、高效地运行，满足用户的各类需求。

本文设计并实现了一款基于SpringBoot和Vue3的智能笔记系统，采用前后端分离架构，完成了用户认证、Markdown在线编辑、分类浏览、全文搜索、评论互动、点赞、收藏及多格式导出等基础功能。在此基础上，系统集成了大语言模型驱动的AI辅助写作模块，提供了内容续写、内容总结、标题优化等智能辅助功能；同时通过知识图谱可视化和写作数据仪表盘实现了对个人知识资产的结构化分析与显示。系统全部数据存储于用户自有数据库，采用JWT无状态认证和BCrypt加密存储机制用来保障用户数据的安全。
经功能测试与接口测试验证，系统各模块运行稳定，核心业务流程均能正常完成，整体达到了预期的设计目标。
通过本系统的开发实践，验证了Vue3+SpringBoot+MyBatis技术组合在个人知识管理场景中的可行性和工程化实施路径。系统在功能完备性、数据自主可控性以及智能化特性方面具备一定优势，可为后续相关系统的开发提供参考。
未来可以进一步拓展：设计或者集成一个更为精确的关键词提取工具，或者添加对于文章内容的关键词提取分析，用来提升知识图谱的关联质量，和扩展更多的基于AI的辅助功能。
